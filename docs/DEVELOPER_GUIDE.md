# 👨‍💻 Руководство разработчика University Portal

Быстрый старт для новых разработчиков проекта.

---

## 🚀 Быстрый старт

### 1. Требования

```bash
node --version    # v20+
pnpm --version    # 8+
docker --version  # 24+
```

### 2. Установка и запуск (Docker Way - Рекомендуется)

Самый быстрый способ развернуть среду со всеми зависимостями (БД, S3, Seeder):

```bash
# Клонирование и переход
git clone <repository-url>
cd university-portal

# Настройка .env
cp .env.example .env

# Запуск всего проекта (включая seeder для наполнения БД)
docker compose up -d --build && docker compose --profile dev up seeder
```

### 3. Локальная разработка

Если нужно запустить только Next.js локально для быстрой отладки:

```bash
cd portal_web
pnpm install
pnpm dev
```

Приложение будет доступно по адресу: **http://localhost:3000**

## 🗄 Работа с базой данных

### Автоматические миграции
В проекте настроена автоматическая синхронизация схемы. При каждом старте Docker-контейнера выполняется:
```bash
npx drizzle-kit push --force
```
Это избавляет от необходимости вручную управлять файлами миграций в процессе активной разработки.

### Заполнение тестовыми данными (Seeder)
Для тестирования пагинации и ролей используйте профиль `dev`:
```bash
# Пересоздает БД и заливает 50 вакансий, 20 студентов и 40 откликов
docker compose --profile dev up seeder --build --force-recreate
```

## 🧩 Особенности реализации

### Back-Navigation
Используйте компонент `BackButton` из `@/shared/ui` для всех переходов назад. Он корректно сохраняет `searchParams` и состояние пагинации через историю браузера.

### Loading States
При работе с серверными данными (фильтры, поиск) используйте:
1.  `loading.tsx` — для первой загрузки страницы.
2.  `useTransition` + `isPending` — для мгновенного отклика интерфейса на изменение фильтров.

### Валидация
Все Server Actions **обязательно** должны валидироваться через `zod`. Схемы валидации расширены для поддержки дат и числовых квот (`availableSpots`).
