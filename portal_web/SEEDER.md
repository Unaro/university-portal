# 🌱 Seeder для University Portal

## Обзор

Seeder автоматически заполняет базу данных и MinIO тестовыми данными для разработки и демонстрации.

## Что делает seeder

### Очистка (перед заполнением)
- 🧹 **База данных**: Удаляет все записи из таблиц в правильном порядке (с учётом FK constraints)
- 🧹 **MinIO**: Очищает bucket от всех файлов

### Заполнение
- 📦 **MinIO**: Загружает тестовые файлы (резюме, материалы, логотипы)
- 📚 **База данных**: Создаёт тестовые данные:
  - 5 специальностей
  - 16 навыков
  - 10 пользователей (админ, преподаватели, представители организаций, студенты)
  - 6 российских компаний (Сбербанк, Яндекс, Газпром, Касперский, Лукойл, МТС)
  - 9 вакансий (включая неоплачиваемые стажировки)
  - 5 студентов с резюме и навыками
  - 9 заявок на вакансии
  - 8 учебных материалов

## Использование

### Автоматически при запуске Docker

Seeder автоматически применяет миграции и заполняет данные при старте:

```bash
docker compose up --build
```

Порядок запуска:
1. Поднимаются PostgreSQL и MinIO
2. Seeded применяет миграции (`drizzle-kit push`)
3. Seeded заполняет БД и MinIO тестовыми данными
4. Запускается веб-приложение

### Ручной запуск seeder в Docker

```bash
docker compose run --rm seeder
```

### Локальная разработка (вне Docker)

Требуется запущенный Docker с PostgreSQL и MinIO:

```bash
# Полное заполнение (с очисткой и миграциями)
pnpm db:seeder

# Только очистка БД (без MinIO)
pnpm db:clear
```

## Переменные окружения

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `CLEAR_DB` | Очищать базу данных перед заполнением | `true` |
| `CLEAR_MINIO` | Очищать MinIO перед заполнением | `true` |

### Примеры

Заполнить БД без очистки:
```bash
CLEAR_DB=false CLEAR_MINIO=false pnpm db:seeder
```

## Решение проблем

### Ошибка подключения к БД
Убедитесь, что PostgreSQL запущен и доступен по hostname `db` (в Docker) или localhost (локально).

### Ошибка подключения к MinIO
Проверьте, что MinIO запущен и доступен по `http://minio:9000` (в Docker) или `http://localhost:9000` (локально).

### Ошибки FK constraints
Seeder удаляет данные в правильном порядке: сначала дочерние таблицы, потом родительские.
