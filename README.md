# University Portal 🎓

Платформа для управления практикой и трудоустройством студентов университета.

## Описание

University Portal — это веб-приложение, которое соединяет студентов, университет и организации-партнёры. Платформа позволяет:

- **Студентам**: просматривать вакансии, отправлять заявки, отслеживать статус откликов
- **Представителям организаций**: публиковать вакансии, управлять откликами
- **Университету**: модерировать организации и вакансии, контролировать трудоустройство

## Технологический стек

### Frontend
- **Next.js 16** — React-фреймворк с App Router
- **TypeScript** — типизация
- **Tailwind CSS 4** — стилизация
- **shadcn/ui** — компоненты интерфейса
- **React Hook Form + Zod** — формы и валидация

### Backend
- **Next.js Server Actions** — серверная логика
- **NextAuth.js (Auth.js)** — аутентификация
- **Drizzle ORM** — работа с базой данных
- **PostgreSQL** — основная БД

### Инфраструктура
- **Docker + Docker Compose** — контейнеризация
- **MinIO** — S3-совместимое хранилище для файлов

### Тестирование
- **Vitest** — тестовый раннер
- **Testing Library** — утилиты для тестирования компонентов
- **Happy-DOM / jsdom** — окружение для тестов

## Быстрый старт

### Требования

- Node.js 20+
- Docker и Docker Compose
- Git

### Установка

1. **Клонировать репозиторий**

```bash
git clone <repository-url>
cd university-portal
```

2. **Установить зависимости**

```bash
cd portal_web
npm install
```

3. **Настроить переменные окружения**

```bash
# Скопировать шаблон
cp .env.example .env

# Отредактировать .env при необходимости
```

4. **Запустить инфраструктуру (PostgreSQL + MinIO)**

```bash
# Из корня проекта
docker-compose up -d
```

5. **Запустить миграции БД**

```bash
cd portal_web
npm run db:push
```

6. **Запустить приложение**

```bash
# Режим разработки
npm run dev

# Продакшен билд
npm run build
npm run start
```

Приложение будет доступно по адресу: http://localhost:3000

## Архитектура

### База данных

Основные сущности:
- **User** — пользователи системы
- **Student** — профили студентов
- **Organization** — организации-партнёры
- **Vacancy** — вакансии от организаций
- **Application** — отклики студентов на вакансии
- **Resume** — резюме студентов
- **Material** — учебные материалы

### Аутентификация

Приложение использует NextAuth.js с email/пароль провайдером. Сессии хранятся в базе данных.

### Хранение файлов

Файлы (логотипы, резюме) загружаются в MinIO (S3-совместимое хранилище).
